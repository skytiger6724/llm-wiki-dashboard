# **一、先把角色分清（这是 90% 人用不好的根源）**
|   |   |   |
|---|---|---|
|工具|在职场中的真实角色|你什么时候该用它|
|NotebookLM|个人事实库 / 复盘中枢 / 判断边界|当你需要「基于过往资料思考」|
|Gemini 3|表达引擎 / 结构整理 / 沟通陪练|当你需要「把话说给别人听」|
铁律一条：
❌ 不要让 Gemini 直接“凭空总结你的一年”
✅ 先让 NotebookLM 看完你的人生资料
# **二、2026 年，你只需要 5 个 NotebookLM 笔记本**
直接照这个结构建：
1️⃣ 年度复盘（2026）

> 年报、季度总结、关键节点材料，只进不删
2️⃣ 项目档案库

> 每个重要项目一个来源集合
3️⃣ 长期问题研究

> 想不明白的问题、零碎判断、反复出现的困惑
4️⃣ 输出母本

> 写过的文章、汇报、方案、PPT 文本
5️⃣ 行业 & 学习资料

> 报告、课程、深度文章（你认可的）
原则：资料一旦进来，就不再靠记忆，全靠对话。
# **三、三类高频职场场景，一套固定用法**
# **场景 1：总结 / 汇报（周报、月报、年报）**
**Step 1｜NotebookLM（事实版）**
基于当前资料，列出我在本周期内实际完成的工作与产出，请避免评价。
**Step 2｜Gemini 3（表达版）**
以下内容是我真实完成的工作，请改写成一份给领导看的总结，强调业务价值，不夸大。
结果：真实 + 好看
# **场景 2：项目 / 方案 / PPT**
**Step 1｜NotebookLM（理解与风险）**
基于历史资料，这个项目成功与失败的关键因素是什么？
**Step 2｜Gemini 3（结构与说服）**
请基于以上分析，生成一份面向管理层的 PPT 大纲（8–10 页）。
结果：你不是在“写方案”，是在“讲判断”
# **场景 3：做选择、提建议（高阶但最值钱）**
**Step 1｜NotebookLM（判断边界）**
基于我过往的资料，在类似问题上我曾犯过哪些判断错误？
Step 2｜Gemini 3（安全表达）
请把以下风险判断，改写成一段适合向领导沟通的表述，理性、不制造焦虑。
结果：你开始像一个成熟决策者
# **四、每天都能用上的「隐形 Prompt」**
# **NotebookLM**
请快速总结这个项目的背景、目标、关键决策和结果，供我对外说明。
# **Gemini 3**
请帮我把以下内容，改写成一段 1 分钟的会议发言，自然、不背稿。
# **五、一个你必须记住的“防翻车清单”**
❌ 不要让 Gemini 编造你没做过的事
❌ 不要把未经 NotebookLM 校验的判断直接往上说
✅ 重要材料，保留 NotebookLM 的原始分析
✅ 对外输出，永远让 Gemini 做最后一道工序
# **六、用一句话总结这套系统的价值**
它不会让你显得更努力，但会让你显得更清楚、更可靠。
在 2026 年，这件事本身，就已经是稀缺能力。
