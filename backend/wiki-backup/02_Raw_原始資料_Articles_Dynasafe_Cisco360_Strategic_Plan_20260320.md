# [[Dynasafe]] Cisco 360 提分戰略全書 (基於 2026/02 現狀分析)

**時間戳記：** 2026-03-20 15:55 (UTC+8)
**分類：** /Users/dwaynejohnson/Documents/00_KM_核心知識庫/Work_Dynasafe/Cisco_360

## 1. 執行大綱
根據 2026 年 2 月份的 Fiscal Month 數據圖表與 Cisco 360 官方 Matrix 進行比對。[[Dynasafe]] 目前平均 PVI 為 6.4 左右，目標應設在 8.5 以上以維持 Preferred 夥伴地位。

## 2. 現狀診斷 (按架構)
*   **Networking (8.50)**：體質穩健。主要缺失：LCI Onboard 執行力不足 (14.89%)。
*   **Security (7.50)**：業績衰退中。主要缺失：TCV YoY 成長負值，且未使用 Buying Program (EA/MSLA)。
*   **Cloud + AI (6.35)**：轉型陣痛期。主要缺失：完全缺乏 AI 業績滲透，ACV 成長嚴重停滯。
*   **Collab (4.80)**：技術斷層。主要缺失：Black Belt 證書數量為零，Capabilities 分數極低。

## 3. 四大提分戰略

### 第一階段：行政與流程補強 (立即生效)
*   **LCI 標記強制化**：由 CS 團隊每週檢查 Networking 與 Security 案件，確保 Onboard 達成率提升至 45% 以上。預計可提升總分 0.5-1.0 分。
*   **Buying Program 滲透**：後續 Security 案件應優先以 Enterprise Agreement (EA) 報價，打破目前 0% 的尷尬紀錄。

### 第二階段：技術證力灌頂 (1-3 個月)
*   **黑帶救援行動**：指派 3 名工程師於一個月內完成 Collaboration Black Belt Stage 1-3 培訓。此舉能讓 Collab 分數從 4.8 衝回 7 分以上。
*   **Managed Services 升級**：申請 Cisco Powered Services 認證，將 MSP 成熟度從 Basic 升至 Expert。這是涵蓋所有架構的通用加分項（Foundational 25%）。

### 第三階段：AI 與 ACV 業績轉向 (長期)
*   **AI 產品組合優化**：在 Cloud 案件中刻意導入 AI Dedicated SKU，目標達成 7% 滲透率，拿滿 Performance 分數。
*   **ACV 成長追蹤**：專注於 Recurring Software 的新合約，扭轉目前負成長的趨勢。

## 4. 結語
[[Dynasafe]] 的技術底子 (Capabilities) 很好，但敗在「代管服務等級」與「系統操作 (LCI/EA)」的細節。只要把 Managed Services 等級拉上來，並落實 LCI 填報，整體 PVI 衝破 9.0 不是問題。
