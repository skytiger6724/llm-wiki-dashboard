## 核心摘要

AppDynamics(Cisco旗下)是企業級 APM 效能監控平台，提供 Core APM、Database Visibility、Server & Network Visibility、Kubernetes & Cloud Native、End User Monitoring、Business IQ 等模組。MLOps 指南說明如何將 ML 模型從實驗階段推向生產環境，強調 DevOps 與 MLOps 的差異。Canonical Kubernetes(MicroK8s)和 Charmed Kubeflow 構成開源 MLOps 堆疊。VMware to the Future 簡報聚焦 Canonical 如何提供開源雲基礎設施替代方案，解決 VMware 授權變更後的挑戰。

## 關鍵概念清單

- **Core APM**：自動端到端可見性、ML 基線 proactive 監控、AI 根因分析加速 MTTR
- **Business IQ**：將應用程式效能映射到商業價值
- **MLOps 生命週期**：獲取數據 → 設計特徵 → 訓練/測試/儲存 → 模型部署 → 監控
- **Charmed Kubeflow**：Canonical 提供的 Kubeflow 管理方案，基於 Juju Operator
- **MicroK8s**：CNCF 認證的輕量 Kubernetes 發行版，適合 Edge 和小型叢集
- **DevOps vs MLOps**：DevOps 處理程式碼+基礎設施；MLOps 額外處理數據、模型、特徵儲存
- **開源四大優勢**：Free、Fast innovation、No vendor lock-in、Lower cost
- **開源採用挑戰**：Rapid upgrade、Frequent update、Lack of support、Documentation
- **Python 依賴管理**：Python 生態系依賴複雜，需要容器化和版本鎖定來確保再現性
- **Ubuntu AD 整合**：Ubuntu Desktop 與 Microsoft Active Directory 整合方案
- **MongoDB 企業資料管理**：文件型 NoSQL 資料庫的企業級部署與治理

## 關鍵實體清單

- **AppDynamics** — Cisco 旗下 APM 平台
- **ThousandEyes** — Cisco 網路效能監控（與 AppDynamics 整合為 FSO）
- **Canonical** — Ubuntu 背後的公司，提供 MicroK8s、Charmed Kubeflow
- **Charmed Kubeflow** — 開源 MLOps 平台
- **MicroK8s** — 輕量 Kubernetes
- **Juju** — Canonical 的 Operator Framework
- **MongoDB** — 文件型 NoSQL 資料庫
- **VMware vSphere / VCF / VVF** — VMware 虛擬化產品線
- **Nutanix** — HCI 競品
- **Ceph** — 開源分散式儲存系統
