# **NotebookLM神级用法：复制这个提示词，三步生成演讲级PPT**

很多人用AI做PPT，出来的东西往往全是车轱辘话，根本没法讲。原因很简单：你把AI当成了“阅读器”，而不是“架构师”。
今天分享一个谷歌神器 **NotebookLM** 的神级用法，配合一套基于 **Prompt Engineering（提示词工程）** 原理设计的**“填空式”模版**。你不需要懂编程，只需要像做填空题一样改几个字，AI就能帮你生成逻辑严密、金句频出的PPT大纲。
---
# **第一步：投喂数据，建立“第二大脑”**
首先，打开 **NotebookLM**，将你的PDF报告、会议记录或技术文档全部上传。
NotebookLM 的强大之处在于它自带 **RAG（检索增强生成）** 能力，它会基于你上传的资料回答问题，而不是在大数据里瞎编。这相当于你雇佣了一个只读过这些文件的超级助手。
---
# **第二步：复制“填空式”模版（核心步骤）**
这是最关键的一步。为了让AI输出高质量内容，我们利用了提示词工程中的**“变量（Variables）”**技术 。
在《Prompt Engineering》白皮书中，谷歌专家指出：**使用变量（如 {city}）可以让你无需重复编写指令，使提示词具有动态性和复用性** 。
我们把复杂的指令封装好，你只需要修改 {} 里的内容：
**立即复制这个 Prompt 到 NotebookLM 对话框：**
```Markdown
# Role: 首席演示架构师 (Chief Presentation Architect)
# Context:
基于上传的所有源文件，运用“认知重塑”原理，将文档中的知识转化为听众大脑容易吸收的“种子”观点，生成一份PPT演讲大纲。
# Variables (请修改以下括号内的内容):
-**{演讲主题}**: [在此输入主题，如：Q3季度营销复盘]
-**{目标听众}**: [在此输入听众，如：公司CEO及投资人]
-**{演讲目标}**: [在此输入目标，如：申请追加下季度预算]
-**{PPT页数}**: [在此输入页数，如：6页]
# Instructions:
1.**全局扫描**: 检索源文件中与 **{演讲主题}** 最相关的核心数据。
2.**听众对齐**: 针对 **{目标听众}** 的痛点筛选信息。
3.**结构化输出**: 严格按照下方 Slide 结构生成。
# Output Format (每页PPT结构):
1.** 标题**: 结论性短句。
2.** 核心洞察**: 一句话解释复杂概念。
3.**️ 画面建议**: 建议配图（图表或比喻）。
4.**️ 演讲备注**: 引用源文件中的具体数据作为支撑。

# **第三步：一键生成，降维打击**
当你填好变量并发送后，NotebookLM 会立即根据你的文档生成一份结构化的演示大纲。
你得到的不再是“第一章、第二章”的废话，而是：
- **针对CEO的**：重点讲回报率、降本增效（因为你在变量里填了听众是CEO）。
- **有画面感的**：它会建议你“左边放锁链图代表高成本，右边放乐高积木代表模块化”（这是因为提示词要求了画面建议）。
- **有数据支撑的**：演讲备注里会自动引用你上传文档的第几页数据。

# **为什么这个模版能“封神”？**
这套模版背后，应用了谷歌《Prompt Engineering》指南中的三个核心心法：
1. **变量思维 (Use Variables)**： 不仅是为了省事。文档指出，通过将特定信息（如主题、听众）存储在变量中，可以避免硬编码，让同一个优秀的提示词逻辑适配无数种场景 。
1. **角色扮演 (Role Prompting)**： 我们在模版开头定义了# Role: 首席演示架构师。这被称为**角色提示（Role Prompting）** 。这能给AI一个“蓝图”，规定它的语气、风格和专业领域，从而生成更专业、更符合身份的输出 。
1. **明确输出结构 (Be specific about the output)**： 我们强制AI按照“标题+洞察+画面+备注”的格式输出。文档强调，明确具体的输出要求（甚至提供JSON或结构化示例），能帮助模型聚焦相关信息，提高准确性 。
